# Возвращаемое значение

## Объект None

Вспомним списки. Иногда нам нужно создать список определенной длины, но у нас еще нет данных для всех элементов.
Например, в соревновании мы знаем бронзового призера, а золото и серебро еще разыгрываются.

```python
winners = [None, None, "Иванов"]
```

Объект `None` (от англ. "ничто") специально создан в Python для обозначения **отсутствия значения** или **пустоты**.

## Новая задача

Давай напишем программу для проверки знаний (викторину).
Нам нужна функция, которая задает вопрос пользователю и проверяет ответ.

Назовем её `give_task`. Она должна принимать два аргумента:
1. Текст вопроса (`question`)
2. Правильный ответ (`correct_answer`)

Примерный алгоритм внутри функции:
1. Спросить у пользователя ответ (используя `input(question)`).
2. Сравнить ответ пользователя с правильным ответом.
3. Если совпало — вывести `Верно!`
4. Если не совпало — вывести `Неверно!`

```python
def give_task(question, correct_answer):
    user_answer = input(question)
    if user_answer == correct_answer:
        print("Верно!")
    else:
        print("Неверно!")

# Проверяем работу
give_task("Столица России? ", "Москва")
give_task("2 + 2 = ? ", "4")
```

### Проблема

Допустим, мы хотим в конце викторины сообщить пользователю, сколько баллов (верных ответов) было набрано. Мы вызываем функцию 3 раза. Но как узнать результат?

```python
points = 0

# задаем вопрос и проверяем ответ
give_task("Столица России? ", "Москва")

# Мы видим "Верно!" или "Неверно!" на экране,
# но переменная points не меняется...
```

Наша функция `give_task` печатает результат на экране, но мы не можем использовать его далее в программе (не можем сохранить в переменной, передать в другую функцию и т.д.)

## Возвращаемое значение (return)

Вспомним аналогию с фабрикой.
*   **Функция** — это цех
*   **Параметры** — это сырье, которое поступает в цех
*   **Возвращаемое значение** — это готовая продукция, результат, произведенный цехом.

Есть цеха, которые производят осязаемые объекты (возвращают значение). А есть цеха, результат работы которых "нематериален" Например отдел, который размещает посты в соцсетях фабрики, пишет новости на официальном сайте. В контейнере для готовой продукции такого отдела ничего нет, там "пусто".
Аналогично с функциями. Одни функции возвращают результат, который можно сохранить в переменную, другие — возвращают "ничего".

### Возвращаемые значения у встроенных функций

Мы уже встречались с функции, которые возвращают результат. Например, функция`len`:

```python
result1 = len("привет")
# В переменную result1 попадет число 6. 
# Функция len "вернула" нам это число, выдала результат
# На экране при этом ничего не появилось
```

А вот функция `print()` работает иначе:

```python
result2 = print("привет")
# В переменной result2 окажется... None
# При этом на экране появится "привет" - это
# и есть результат работы функции print
```

Функция `print` печатает текст на экран (это её работа), но **возвращает** пустоту (`None`) - она так реализована.

### Инструкция return

Чтобы наша функция `give_task` не просто печатала, но и "отдавала" результат (баллы), нужно использовать инструкцию `return`.

```python
def give_task(question, correct_answer):
    points = 0
    user_answer = input(question)
    if user_answer == correct_answer:
        print("Верно!")
        points += 1
    else:
        print("Неверно!")
    return points  # Возвращаем кол-во набранных баллов
```

Теперь мы можем использовать результат работы функции!

```python
answer1 = give_task("Столица России? ", "Москва")
answer2 = give_task("2 + 2 = ? ", "4")
answer3 = give_task("Зимой и летом одним цветом? ", "Елка")

print("Ваш счет:", answer1 + answer2 + answer3)
```

Когда Python встречает `return`, функция **завершается** и на месте ее вызова в основной программе возникает значение, которое она вернула (результат ее работы).

## Скрытый return

А как же наши функции работали раньше без `return`?
Оказывается, если в функции нет инструкции `return`, Python сам (автоматически и незаметно для нас) добавляет в самый конец строчку:

```python
return None
```

**Вывод:**
1. Если функция должна что-то обработать и отдать результат для дальнейшего использования — используем `return`.
2. Если функция взаимодействует с внешним миром (экраном, жестким диском, сетью и т.д.) `return` можно не писать (Python допишет `returnNone` автоматически).

### Задание: Магазин

Ты пишешь программу для кассы магазина.
У тебя есть функция `buy_product`, которая оформляет покупку.

**Часть 1 (без return)**
Сейчас функция выглядит так. Она принимает название товара и цену, и просто печатает чек.

```python
def buy_product(product, price):
    print(f"Вы купили {product} (цена в монетах: {price})")

print("Добро пожаловать в магазин!")
buy_product("Яблоко", 10)
buy_product("Меч", 50)

```

**Проблема:**
У покупателя в кошельке было 100 монет. Нам нужно вычитать стоимость покупок из кошелька. Но сейчас функция `buy_product` не сообщает программе цену товара, она только печатает ее на экране пользователя.

**Твоя задача:**

1. Перепиши функцию `buy_product` так, чтобы она **возвращала** цену купленного товара.
2. В основной программе создай переменную `wallet = 100` (кошелек).
3. Вызови функцию покупки 3 раза (купи разные товары).
4. После каждой покупки уменьшай количество денег в `wallet` на ту сумму, которую вернула функция.
5. В конце напечатай остаток в кошельке.

**Пример (логика):**

```bash
Ваш баланс: 100
Вы купили Яблоко (цена в монетах: 10)
В кошельке осталось: 90
Вы купили Меч (цена в монетах: 50)
В кошельке осталось: 40
```
