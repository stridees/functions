# Параметры и аргументы (часть 2)

Мы уже умеем передавать текст рекламы в функцию.
Но заказчик стал более требовательным. Теперь он хочет менять не только текст, но и **оформление**.
В одних объявлениях он хочет видеть волнистые линии `~~~`, а в других — собачки `@@@` или звездочки `***`.

## Несколько параметров

Чтобы задать функции и текст, и стиль оформления, нужно передать ей **два** значения.
Для этого в объявлении указывается два параметра через запятую:

```python
def show_ads(text, sticker):
    print()
    print(sticker)  # Верхняя граница
    print(text)     # Текст рекламы
    print(sticker)  # Нижняя граница
    print()

# Вызов 1: реклама с волнистыми линиями
show_ads("Купите наших котиков!", "~~~")

# Вызов 2: реклама с знаками собаки
show_ads("Только сегодня скидки!", "@@@")
```

## Позиционные аргументы (порядок важен!)

Python не понимает смысла слов "text" или "sticker". Он просто учитывает порядок - как параметры расположены в скобках, то есть их позицию. Такие параметры (и соответствующие им аргументы) называются **позиционными**.

Первое значение в скобках (`"Купите..."`) становится значением переменной `text`. Второе значение (`"~~~"`) будет присвоено переменной `sticker`.

**Что будет, если перепутать?**

Представь, что мы вызвали функцию так:

```python
show_ads("@@@", "Только сегодня скидки!")
```

Для компьютера здесь нет ошибки: он видит два объекта и подставляет их:
- `text` станет `"@@@"`
- `sticker` станет `"Только сегодня скидки!"`

Но на экране получится следующее:
```
Только сегодня скидки!
@@@
Только сегодня скидки!
```
Это **логическая ошибка**. Программа работает, но делает не то, что нужно.

## Обязательные аргументы

Если мы объявили функцию с двумя параметрами, мы **обязаны** передать два значения (аргумента).

```python
show_ads("Яндекс - лучший поиск!")
# ОШИБКА! TypeError: ... missing 1 required positional argument: 'sticker'
```

В прошлом задании мы реализовали воспроизведение разных мелодий в функции `play_music` (она принимала один аргумент)
Теперь диджей хочет управлять **длительностью** джингла. Иногда нужно сыграть "Ля" 3 раза, а иногда — 5 или 7 раз.

**Твоя задача:**

1. Изменить функцию `play_music` так, чтобы она при вызове принимала **два** аргумента (_что_ играть и _сколько раз_)   
2. Вызови функцию несколько раз с разными значениями.

Пример вызова:
```python
play_music("Пам", 2)
# Должно вывести:
# Пам
# Пам
```

После того как напишешь правильный код, попробуй специально перепутать аргументы местами при вызове:

```python
# Передаем сначала число, потом строку
play_music(3, "До")
```

**Подумай:**
* Почему в примере с рекламой (`text`, `sticker`) перестановка мест меняла смысл, но программа работала?
* Почему в примере с музыкой (`melody`, `count`) перестановка мест приведет к **аварийной остановке** программы (ошибке)?

<details>
<summary>Подсказка про ошибку</summary><br>

В функции с музыкой используются **разные типы данных**: строка и число.
Функция ждет число для `range()`, а мы передадим ей строку. Функция ждет строку для `print()`, а получит число. 
Python не сможет выполнить `range("Пам")` и отобразит сообщение об ошибке.

Вспомним аналогию с цехом на фабрике. Если цех производит хлеб, а в качестве сырья предоставили гвозди, то в работе цеха возникнет сбой.


**Вывод:** Важно следить за тем, **ЧТО** мы передаем в функцию
</details>

def play_music(melody, count):
